## 1. What is Cupcake

Cupcake is a multi-channel AI assistant built on the **OpenClaw** runtime framework. She operates across Telegram, phone (voice), web dashboard, and the OpenClaw CLI. She has a distinct personality ("princess of apocalypse, coder goddess, zero-assumption queen"), access to 10+ AI models via OpenRouter, and is wired into a full automation ecosystem via n8n (email, calendar, contacts, voice synthesis, deployment).

**Core Stack:**
- **Runtime:** OpenClaw 2026.2.13
- **Primary AI:** DeepSeek R1 (reasoning) via OpenRouter
- **Fast AI:** DeepSeek V3.2 (voice/quick responses) via OpenRouter
- **Image Gen:** Nano Banana Pro (Gemini 3 Pro), Seedream, GPT-5 Image Mini
- **Voice:** Vapi.ai with ElevenLabs Savannah voice
- **Automation:** n8n (30 workflows on automate.hales.ai)
- **Hosting:** Digital Ocean droplet (134.199.239.171)

### Security Measures
- `.env` file has `chmod 600` permissions
- `.gitignore` excludes `.env`, `memory/`, `.openclaw/`
- Telegram bot locked to owner's user ID only (configured in OpenClaw)
- OpenClaw gateway binds to loopback only (not internet-accessible)
- Web dashboard requires password authentication
- GitHub PAT is fine-grained with minimal scope (Contents + PR, single repo)
- n8n has separate login credentials from the API key

### Security Audit Results (Feb 14, 2026)

**CRITICAL:**
- UFW (firewall) is **inactive** on the droplet. All bound ports (3000, 3456, 5678, 8001, 8003, 8080, 8082, 8090, 9000) are directly internet-accessible, bypassing nginx/SSL. **Action:** Enable UFW with rules for 22, 80, 443 only.
- n8n (port 5678) is bound to all interfaces â€” accessible directly without SSL at `http://134.199.239.171:5678`. This bypasses authentication.

**IMPORTANT:**
- Rotate the GitHub PAT before it expires (7-day window from Feb 14)
- All API keys validated as NOT committed to any GitHub repo (audit passed)
- Web dashboard password is hardcoded in the React source â€” consider environment variable
- n8n webhook endpoints have no rate limiting â€” vulnerable to abuse

**RECOMMENDED:**
- Enable UFW: `ufw allow 22,80,443/tcp && ufw enable`
- Bind n8n to 127.0.0.1 only (nginx handles external access)
- Upgrade web dashboard auth to OAuth or magic link
- Add rate limiting to n8n webhooks (via nginx or n8n config)
- Rotate OpenRouter API key periodically
- Set up automated SSL cert renewal monitoring
